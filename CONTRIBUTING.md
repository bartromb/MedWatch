### How can I contribute?

**Report a bug**
1. Check if the issue already exists under [Issues](../../issues)
2. Open a new issue with:
   - Browser and version
   - Steps to reproduce
   - Expected vs actual behaviour
   - Screenshot if applicable

**Suggest a feature**
- Open an issue with the `enhancement` label
- Describe the use case and why it would be valuable

**Submit code**
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Edit `index.html` â€” everything lives in this one file
4. Test in multiple browsers (Chrome, Firefox, Safari, Edge)
5. Commit with a clear message: `git commit -m 'Add: description'`
6. Submit a Pull Request to the `main` branch

### Style guide

- **No build step** â€” keep the app as a single HTML file
- **No new external dependencies** â€” only use what is already loaded (React, Babel)
- **CSS variables** â€” use existing `:root` custom properties for colours
- **Dutch UI text** â€” user-facing strings are Dutch; code and comments may be English
- **Commit messages** in English, format: `Add: ...` / `Fix: ...` / `Change: ...`
